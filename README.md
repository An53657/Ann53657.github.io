# 博客项目技术架构说明

## 一、整体技术栈与设计风格

本项目采用 **HTML5 + CSS3 + 原生 JavaScript ES6语法** 进行前端构建，引入 **Bootstrap 4** 框架实现全端响应式适配，确保网站在桌面端、平板端和手机端均能呈现一致且良好的视觉效果与交互体验。

设计风格上融合了 **Material Design 3 (MD3)** 的现代简洁美学与猫娘乐园 (Nekopara) 的软萌治愈元素，打造出兼具设计感与个性化的博客界面。

## 二、核心设计理念

项目严格遵循 **数据与视图分离** 的架构原则，最大限度降低代码耦合度：

- 所有页面内容数据统一存储于独立文件中，HTML 文件仅负责页面结构骨架的定义
- 该设计为后续接入 MySQL 等关系型数据库实现动态数据管理预留了完整的扩展空间，未来只需修改数据获取层逻辑即可完成静态到动态网站的平滑迁移

## 三、模块化文件结构

项目采用高度模块化的文件组织方式，将不同职责的代码拆分到独立文件中，便于后续维护与功能扩展：

### 1. 入口文件

- `index.html`：网站主入口文件，仅包含页面的基本 HTML 骨架，不内嵌任何样式与业务逻辑代码

### 2. 数据层

- `data.json`：统一存储所有页面数据（包括文章信息、个人资料、公告内容等），作为前端与后端数据库之间的过渡层

### 3. 样式层

样式文件按职责拆分为三个独立文件，实现样式的分层管理：

- `base.css`：网站基础样式文件，包含全局 CSS 重置、主题配色系统、基础字体与间距定义
- `main.css`：全局通用样式文件，当前主要承载首页的整体布局与核心样式
- `components.css`：可复用组件样式文件，统一管理文章卡片、导航栏、侧边栏信息卡、按钮等所有独立组件的样式

### 4. 脚本层

所有 JavaScript 业务逻辑代码均拆分至独立的脚本文件中，与 HTML 结构和 CSS 样式完全分离，提升代码的可维护性与复用性。



## 四、未来技术架构升级规四

为实现项目的动态化、工程化与可扩展性，未来将逐步引入以下技术栈完成架构升级：

### 1. 前端架构升级：Vue.js 生态

- **目标**：将当前静态 HTML 页面重构为基于 Vue.js 的单页应用 (SPA)，实现组件化开发与高效的视图更新

- **核心规划**：

  - 使用 Vue CLI/Vite 搭建现代化前端工程
  - 将现有`components/`下的 HTML 模块重构为 Vue 单文件组件 (SFC)
  - 引入 Vue Router 实现页面路由管理，替代静态页面跳转
  - 保留并适配现有的 MD3 + Nekopara 风格 CSS 样式

  

### 2. 前后端数据交互：Ajax/Axios

- **目标**：实现前后端分离，替代当前`data.js`的静态数据模式

- **核心规划**：

  - 引入 Axios 库作为 HTTP 客户端，统一管理 API 请求
  - 设计 RESTful API 接口规范，为后续 Java 后端对接预留标准
  - 实现数据的动态加载、提交与状态管理（如文章列表分页、评论功能）
  - 保留`data.json`作为过渡性 Mock 数据，确保开发阶段的前后端并行

  

### 3. 后端服务建设：Java 生态

- **目标**：构建稳定、可扩展的后端服务，替代当前纯前端静态架构

- **核心规划**：

  - 基于 Spring Boot 框架搭建 Java 后端应用
  - 集成 MySQL 数据库，设计文章、用户、评论等核心数据表
  - 实现 MyBatis/JPA 数据持久层，完成数据的 CRUD 操作
  - 提供标准化的 JSON API 接口，与前端 Axios 请求无缝对接
  - 预留用户认证、权限管理等扩展接口

  

### 4. 容器化部署：Docker

- **目标**：实现开发、测试、生产环境的一致性，简化部署流程

- **核心规划**：

  - 编写 Dockerfile，分别构建前端 Vue 应用与后端 Java 应用的镜像
  - 使用 Docker Compose 编排前端、后端、MySQL 数据库等多容器服务
  - 实现容器化的一键部署与版本更新
  - 为后续 CI/CD 流程（如 GitHub Actions/Jenkins 集成）预留接入点

## 项目结构
```bash
Blog_Project/
├── assets/                 # 静态资源
│   ├── css/
|   |   |—— base.css        # 基础样式
│   │   ├── main.css        # index.html的样式
│   │   └── components.css  # 局部组件样式（如猫咪卡片、导航栏）
│   ├── js/
│   │   └── data.js         # 模拟数据库，存放你要“写死”的文章、图片信息
|   |   |—— main,js         # 全局JS
│   ├── img/
│   │   ├── avatars/        # 头像
│   │   ├── banners/        # 封面图
│   │   └── ui/             # 装饰性图标（猫爪、铃铛等）
│   └── lib/                # 第三方库（Bootstrap 源码或本地备份）
├── components/             # 页面公共模块（虽然是静态，但逻辑上独立）
│   ├── navbar.html         # 导航栏构思
│   └── footer.html         # 页脚构思
├── pages/                  # 独立页面
│   ├── archive.html        # 归档页
│   ├── about.html          # 关于我
│   └── post-detail.html    # 文章详情页
└── index.html              # 首页入口
```
